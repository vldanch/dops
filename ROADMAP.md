# Roadmap for dops

This roadmap outlines planned features and improvements for the dops CLI tool — a DevOps assistant written in Go.

---

## Stage 1 — Basic functionality (DONE / In Progress)
- [x] `ping` command: check HTTP service availability
- [x] `checksystem` command: CPU, memory, disk, uptime info
- [x] Support for configuration file (`config.yaml`)
- [ ] Add output formatting (JSON, plain text)
- [ ] Add unit and integration tests

## Stage 2 — Extended DevOps utilities
- [ ] `checkport` command: check TCP port availability
- [ ] `checkdns` command: DNS resolution checks
- [ ] `notify` command: send notifications (Telegram integration)
- [ ] Improved logging and error handling

## Stage 3 — Packaging and distribution
- [ ] Build binaries for major platforms (Linux, Windows, MacOS)
- [ ] Create Docker container image
- [ ] Publish releases on GitHub with changelog

## Stage 4 — Advanced features
- [ ] Web UI dashboard to monitor checks
- [ ] REST API to trigger commands remotely
- [ ] Plugin system to allow custom commands

## Stage 5 — Documentation and community
- [ ] Detailed user guide and examples
- [ ] Contribution guidelines
- [ ] Public roadmap and issue tracker management
- [ ] Promote project and gather feedback
